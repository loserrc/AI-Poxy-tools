【解决方案 - 请按顺序执行】

===============================================================================
步骤 1: 安装 Root CA 证书
===============================================================================

1.1 以管理员身份打开 PowerShell
    - 按 Win + X 键
    - 选择 "Windows PowerShell (管理员)" 或 "终端 (管理员)"
    - 点击"是"允许

1.2 切换到项目目录
    在 PowerShell 中输入：
    cd D:\Users\ASUS\Desktop\trae-poxy
    按回车

1.3 安装证书
    在 PowerShell 中输入：
    certutil -addstore root certs\ca.pem
    按回车

1.4 验证结果
    应该看到：
    "CertUtil: -addstore 命令成功完成"

    如果看到错误，请记录错误信息。

1.5 刷新 DNS
    在同一个 PowerShell 中输入：
    ipconfig /flushdns
    按回车

===============================================================================
步骤 2: 配置 Trae
===============================================================================

2.1 打开 Trae 应用程序

2.2 进入设置
    - 查找"设置"、"配置"、"Settings"或"Preferences"选项
    - 通常在菜单栏或右上角

2.3 找到 API 端点配置
    - 查找以下名称的选项：
      * API 端点
      * Base URL
      * API URL
      * 服务器地址
      * Server Address
      * Endpoint

2.4 修改 API 端点
    将当前的地址改为：
    https://api.openai.com:8443

    重要：必须包含 :8443 端口号！

2.5 保存设置
    - 点击"保存"、"应用"或"确定"按钮

===============================================================================
步骤 3: 重启 Trae
===============================================================================

3.1 完全关闭 Trae
    - 不是最小化到托盘
    - 如果 Trae 在系统托盘，右键点击图标，选择"退出"
    - 或者在任务管理器中结束 Trae 进程

3.2 等待 5 秒

3.3 重新启动 Trae

3.4 等待 Trae 完全启动

===============================================================================
步骤 4: 测试
===============================================================================

4.1 在 Trae 中发送一条简单的测试消息
    例如："你好"

4.2 观察结果
    - 如果消息正常返回 → 成功！问题已解决
    - 如果出现错误 → 记录错误信息

===============================================================================
步骤 5: 验证代理是否收到请求
===============================================================================

5.1 打开文件：
    D:\Users\ASUS\Desktop\trae-poxy\logs\trae_poxy.log

5.2 查看最后几行
    - 如果看到 "incoming method=POST" → 代理收到了请求
    - 如果没有 → Trae 还没有连接到代理

===============================================================================
常见问题和解决方案
===============================================================================

问题 1: 找不到 Trae 的设置
解决：
- 查看 Trae 的菜单栏
- 查看右上角的齿轮图标或三个点的菜单
- 查看"工具"、"选项"、"首选项"等菜单项

问题 2: 找不到 API 端点选项
解决：
- 查找"高级设置"或"Advanced Settings"
- 查找"网络"或"Network"设置
- 查找"API"或"服务器"相关设置

问题 3: 修改后仍然失败
可能原因：
a) 证书未正确安装
   - 重新执行步骤 1
   - 验证命令：certutil -store root | findstr "Trae-Poxy"
   - 应该有输出

b) Trae 没有重启
   - 必须完全关闭并重新启动 Trae

c) API 端点配置错误
   - 确认配置的是：https://api.openai.com:8443
   - 注意：必须有 :8443

d) 端口 8443 被防火墙阻止
   - 检查 Windows 防火墙设置
   - 允许端口 8443 的入站连接

问题 4: 仍然出现 4028 错误
这说明：
- Trae 绕过了代理，直接连接到上游服务器
- 需要检查 Trae 的 API 端点配置是否正确
