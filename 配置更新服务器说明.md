# AI Poxy Tools 更新服务器配置说明

## 版本信息
- **当前版本**: v1.2.0
- **打包日期**: 2026-02-13

## 如何配置更新服务器地址

### 方法 1：修改 .env 文件（推荐）

1. **在 EXE 同目录创建 `.env` 文件**：
   ```
   dist\AI-Poxy-Tools.exe 同目录
   ```

2. **编辑 `.env` 文件内容**：
   ```env
   APP_NAME=AI Poxy Tools
   APP_ICON=icons/logo.png
   APP_VERSION=v1.2.2
   APP_DEVELOPER=Ying XingYao
   APP_WEBSITE=https://newapi.loserrc.com/
   APP_PROJECT_URL=https://github.com/loserrc/AI-Poxy-tools

   # 更新服务器配置
   UPDATE_CHECK_URL=https://raw.githubusercontent.com/loserrc/AI-Poxy-tools/main/latest.json
   ```

3. **保存文件**，重启应用即可生效

### 方法 2：使用 COLLECT 文件夹版本

如果使用 `dist\AI-Poxy-Tools\` 文件夹版本：
- `.env` 文件已经打包在文件夹根目录
- 直接编辑 `dist\AI-Poxy-Tools\.env` 即可

## 更新服务器 JSON 格式

在你的服务器上创建 `latest.json` 文件，格式如下：

```json
{
  "version": "v1.3.0",
  "download_url": "https://github.com/loserrc/AI-Poxy-tools/releases/download/v1.2.2/AI-Poxy-Tools.exe",
  "changelog": "v1.3.0 更新内容：\n- 新功能 1\n- Bug 修复 2\n- 性能优化 3"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `version` | string | 是 | 最新版本号（格式：v1.2.3） |
| `download_url` | string | 是 | 下载链接（完整 URL） |
| `changelog` | string | 是 | 更新日志（支持 \n 换行） |

## 使用更新检测功能

1. **启动应用**（确保已配置 `UPDATE_CHECK_URL`）
2. **点击菜单**：帮助 → 检查更新
3. **查看结果**：
   - 如果有新版本：显示版本对比、更新日志和下载链接
   - 如果已是最新：提示"当前已是最新版本"
   - 如果网络错误：提示"无法连接到更新服务器"

## 部署更新服务器

### 示例 1：使用 Nginx 静态托管

```nginx
server {
    listen 443 ssl;
    server_name newapi.loserrc.com;

    location /ai-poxy-tools/latest.json {
        root /var/www/html;
        add_header Content-Type application/json;
        add_header Access-Control-Allow-Origin *;
    }

    location /ai-poxy-tools/download/ {
        root /var/www/html;
        add_header Content-Disposition "attachment";
    }
}
```

文件结构：
```
/var/www/html/ai-poxy-tools/
├── latest.json
└── download/
    ├── Trae-Poxy-UI-v1.2.0.exe
    └── Trae-Poxy-UI-v1.3.0.exe
```

### 示例 2：使用 GitHub Releases

1. **创建 GitHub Release**
2. **上传 EXE 文件**
3. **创建 latest.json**：
   ```json
   {
     "version": "v1.2.0",
     "download_url": "https://github.com/loserrc/AI-Poxy-tools/releases/download/v1.2.2/AI-Poxy-Tools.exe",
     "changelog": "详见 GitHub Release 页面"
   }
   ```
4. **配置 UPDATE_CHECK_URL**：
   ```
   UPDATE_CHECK_URL=https://raw.githubusercontent.com/loserrc/AI-Poxy-tools/main/latest.json
   ```

### 示例 3：使用 CDN

```env
UPDATE_CHECK_URL=https://cdn.example.com/ai-poxy-tools/latest.json
```

## 常见问题

### Q1: 提示"未配置更新服务器地址"
**A**: 检查 `.env` 文件是否存在，且 `UPDATE_CHECK_URL` 是否正确配置

### Q2: 提示"无法连接到更新服务器"
**A**:
- 检查网络连接
- 确认 URL 可访问（浏览器测试）
- 检查防火墙设置

### Q3: 提示"更新信息格式错误"
**A**:
- 检查 JSON 格式是否正确
- 确认包含必填字段：version, download_url, changelog

### Q4: 版本号比较规则
**A**:
- 支持格式：`v1.2.3` 或 `1.2.3`
- 比较规则：主版本 > 次版本 > 修订版本
- 示例：`v1.3.0` > `v1.2.9` > `v1.2.0`

## 版本发布流程建议

1. **开发完成** → 更新 `.env` 中的 `APP_VERSION`
2. **重新打包** → `pyinstaller trae_poxy_ui.spec --clean`
3. **上传文件** → 上传 EXE 到服务器/GitHub
4. **更新 JSON** → 修改 `latest.json` 的版本和下载链接
5. **测试验证** → 使用旧版本测试更新检测

## 安全建议

1. **使用 HTTPS**：确保 `UPDATE_CHECK_URL` 使用 HTTPS 协议
2. **验证签名**：建议对 EXE 文件进行数字签名
3. **CDN 加速**：使用 CDN 提高下载速度
4. **版本回退**：保留历史版本以便回退
